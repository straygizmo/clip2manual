# clip2manual 設計ドキュメント

- 日付: 2026-05-26
- ステータス: 設計確定（実装計画作成前）

## 目的

デスクトップをナレーション付きでレコーディングした動画素材を、見やすいマニュアル動画に
自動変換するデスクトップアプリ。以下を実現する。

- クリックしたポイントの強調表示（リップル / 波紋）
- ナレーションの文字起こし → 不自然な点を LLM で修正 → VOICEVOX 音声で元音声を差し替え
- タイムライン表示
- 音声（セグメント）ごとの TTS やり直し
- ビデオ編集ソフトのような UI（クラシック NLE 型）

## 確定した主要な意思決定

| 項目 | 決定 |
|------|------|
| 技術スタック | Electron + TypeScript + React |
| 入力 | アプリ自身で画面録画（クリック座標・時刻・ナレーション音声を同時記録） |
| 文字起こし | ローカル（whisper.cpp、モデル同梱、オフライン） |
| LLM 補正 | クラウド。プロバイダ抽象化で Anthropic (Claude) / OpenAI / Azure OpenAI を切替可 |
| TTS | VOICEVOX（ローカル HTTP、初回起動時に自動ダウンロード） |
| 利用形態 | 社内・チーム共有（非技術者も使う。セットアップは極力自動） |
| 編集範囲 | 軽量編集（タイムライン・セグメント編集・再TTS・カット/トリム） |
| クリック強調 | リップル（波紋） |
| 画面レイアウト | クラシック NLE 型（中央プレビュー / 右インスペクタ / 下タイムライン） |
| タイミング調整 | 映像を音声に合わせる（音声が長ければフリーズ保持、短ければ末尾に小休止） |
| VOICEVOX 配信 | 初回起動時に自動ダウンロード（アプリが取得・起動を管理） |

## ① 全体アーキテクチャ

```
Renderer プロセス (React UI)
  ・編集画面（プレビュー / インスペクタ / タイムライン）
  ・プレビューはリアルタイム合成（書き出しまで非破壊）
        │ IPC
Main プロセス (Node.js / オーケストレーション)
  ・画面録画の制御（desktopCapturer）
  ・グローバルマウスフック（uiohook-napi）
  ・子プロセス/サービス呼び出し ↓
   ├─ whisper.cpp（ローカル）
   ├─ LLMService（クラウド: Anthropic / OpenAI / Azure）
   ├─ VOICEVOX engine（ローカル HTTP）
   └─ FFmpeg（書き出し）
```

役割分担:
- Renderer は UI とプレビュー再生のみ。重い処理は持たない。
- Main がすべての外部プロセス/サービスをオーケストレーション。重い処理（文字起こし・
  書き出し）は子プロセスや worker で実行し UI をブロックしない。
- 各処理エンジンは疎結合の独立モジュールとして実装し、個別にテスト可能にする:
  `RecorderService` / `TranscriptionService` / `LLMService` / `TTSService` / `ExportService`。

### LLM プロバイダ抽象化

`LLMService` を共通インターフェースとし、各プロバイダ（Anthropic / OpenAI / Azure OpenAI）を
差し替え可能な実装にする。エンドポイント・モデル・APIキーは設定で管理。

## ② 録画の仕組み

録画時に 3 種類のデータを共通の開始時刻 (t0) を基準に同時記録する。

| 記録対象 | 手段 | 出力 |
|---------|------|------|
| 画面映像 | desktopCapturer で画面/ウィンドウ選択 → getDisplayMedia → MediaRecorder | 映像（webm → 後段で mp4 化） |
| ナレーション音声 | getUserMedia（マイク）を別トラックとして記録 | 独立した音声ファイル |
| クリックログ | uiohook-napi のグローバルマウス押下イベント | `{x, y, button, t_ms}` の配列（t0 相対） |

技術ポイント — 座標変換:
クリック座標は OS の物理ピクセルで取得されるが、録画映像のピクセル座標とは DPI スケーリングや
マルチモニタ環境でズレる。録画対象ディスプレイの位置・解像度・スケール係数を記録時に取得し、
「OS座標 → 映像内ピクセル座標」へ変換する。誤ると強調表示位置がズレるため慎重に実装＋テストする。

同期: 音声・映像・クリックすべてを t0 からの相対ミリ秒で管理。プレビューと書き出しで同一の
タイムベースを使う。

MVP スコープ: 強調表示はクリック点のみ（カーソルの常時追跡はしない）。

## ③ 処理パイプライン

録画後、自動で以下を実行。各ステップの結果は保存され、後から編集可能。

1. 文字起こし（whisper.cpp）: セグメント配列 `[{start, end, text}, ...]`。各セグメントが
   編集の最小単位。
2. LLM 補正（LLMService）: 各セグメントの「自然な文」に修正。元文と修正文の両方を保持
   （差分表示可）。
3. TTS 生成（VOICEVOX）: 修正文ごとに音声クリップを合成。声・速度はセグメント単位で上書き可。
4. 差し替え & タイミング調整: 「映像を音声に合わせる」をデフォルトとする。音声が長ければ
   最終フレームを静止保持（フリーズ）、短ければ末尾に小休止。

## ④ データモデルとプロジェクト保存

すべて非破壊編集。元データは常に保持し、編集内容はデータとして持ち、書き出し時に焼き込む。

プロジェクト = マニフェスト (JSON) + アセットフォルダ。

```jsonc
// project.json
{
  "meta": { "name": "...", "createdAt": "...",
            "source": { "video": "raw.webm", "narration": "narration.wav",
                        "clickLog": "clicks.json",
                        "display": { "width": 2560, "height": 1440, "scale": 1.25 } } },
  "settings": { "highlightStyle": "ripple", "timingMode": "video-follows-audio",
                "llm": { "provider": "anthropic", "model": "..." },
                "tts": { "defaultSpeaker": 3, "defaultSpeed": 1.0 } },
  "segments": [
    {
      "id": "seg-001",
      "videoStart": 0.0, "videoEnd": 4.2,
      "originalText": "ここをくりっくします",
      "correctedText": "ここをクリックします",
      "ttsAudio": "tts/seg-001.wav",
      "voice": { "speaker": 3, "speed": 1.0 },
      "clicks": [ { "x": 980, "y": 540, "t": 1.8 } ],
      "enabled": true
    }
  ]
}
```

ポイント:
- セグメントが編集の中心単位。結合・分割・無効化（カット）が可能。
- 動画の指定区間の削除は、対象セグメントの `videoStart`/`videoEnd` をトリム、または分割・除去
  として表現する（元映像は保持し、編集データ側で区間を持つ非破壊方式）。
- 元テキストと修正テキストを両方保持 → 差分表示・元に戻すが可能。
- TTS クリップはセグメントごとにファイル化 → 個別の再生成が安価。
- アセットフォルダに元映像・元音声・生成 TTS を格納。プロジェクトを開き直せば編集を継続できる。

## ⑤ 編集 UI・プレビュー・書き出し

編集 UI（クラシック NLE 型）:

```
ツールバー: ● 録画 | ▶ 書き出し | プロジェクト名 | ⚙ 設定
─────────────────────────────────┬──────────────────────────
                                  │ インスペクタ（選択中セグメント）
       プレビュー（中央・大）         │ ・テキスト編集（元 vs 修正の差分）
  元映像＋リップル合成をリアルタイム    │ ・声の選択 / 速度
  再生（書き出しまで非破壊）          │ ・[再TTS] ボタン
                                  │ ・クリック強調の位置/有無
                                  │ ・セグメント有効/無効（カット）
─────────────────────────────────┴──────────────────────────
 タイムライン
  [映像        ▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬ ]
  [クリック強調  ◆      ◆    ◆      ]
  [ナレーション ▮▮ ▮▮▮ ▮▮  ▮▮▮▮ ▮▮ ]  ← クリックで選択 → インスペクタ更新
```

タイムライン編集操作:
- 動画の指定区間の削除: タイムライン上で範囲（開始〜終了）を選択して映像ごと削除し、後続を
  詰める。削除範囲に重なるセグメントはトリム/分割し、含まれるクリック強調・TTS も連動して
  除去・再配置する。すべて非破壊（元データは保持し編集データとして記録）。
- セグメントの結合・分割、端のトリム。

プレビュー（リアルタイム合成・非破壊）:
- 映像は `<video>`/canvas で再生し、クリック時刻にリップルを canvas オーバーレイで描画
  （変換済み座標を使用）。
- 音声は Web Audio API で各セグメントの TTS クリップをスケジュール再生（元音声/TTS をトグル可）。
- 「映像を音声に合わせる」はプレビューでもフリーズ保持を再現。

書き出し（ExportService → FFmpeg）:
1. セグメントごとに映像をリタイミング（フリーズフレーム挿入 / 末尾小休止）。
2. リップルアニメーションの焼き込み — FFmpeg の overlay フィルタはアニメーションが苦手なため、
   リップルを透過オーバーレイ（PNG シーケンス or アルファ付き動画）として canvas でレンダリングし、
   FFmpeg で base 映像に合成する方式を採用。
3. 全セグメントの TTS 音声を連結して多重化。
4. 出力: MP4（H.264 + AAC）、元解像度。

## ⑥ セットアップと同梱（チーム共有・簡単導入）

初回起動ウィザードで必要なものを自動的に整える。

| 要素 | 方針 |
|------|------|
| whisper.cpp | バイナリ＋日本語モデル（ggml）をアプリに同梱。子プロセスとして自動実行。オフライン動作 |
| VOICEVOX エンジン | ローカル HTTP（localhost:50021）。初回起動時に自動ダウンロード・起動をアプリが管理 |
| LLM API キー | 設定画面で入力。Electron safeStorage（OS キーチェーン）で暗号化保存。プロバイダも設定で選択 |
| インストーラ | electron-builder（Windows NSIS） |
| 初回ウィザード | 依存関係の確認・API キー入力・デフォルト話者選択をガイド |

ライセンス注意:
- VOICEVOX は生成音声に話者ごとのクレジット表記が必要。書き出し時／アプリ内に明記する。
- whisper.cpp および同梱モデルのライセンスを確認し遵守する。

## モジュール構成（疎結合）

| モジュール | 責務 | 主な依存 |
|-----------|------|---------|
| RecorderService | 画面・音声・クリックの同時記録、座標変換 | desktopCapturer, uiohook-napi |
| TranscriptionService | 音声 → セグメント配列 | whisper.cpp |
| LLMService | テキスト補正（プロバイダ抽象化） | Anthropic / OpenAI / Azure |
| TTSService | テキスト → 音声クリップ | VOICEVOX engine |
| ExportService | リタイミング・オーバーレイ合成・多重化・書き出し | FFmpeg |
| ProjectStore | プロジェクト JSON とアセットの読み書き | fs |

## 段階的実装の想定（フェーズ）

1. 録画基盤（RecorderService）＋座標変換＋プロジェクト保存
2. 文字起こし（TranscriptionService）＋セグメントのタイムライン表示
3. LLM 補正（LLMService）＋テキスト編集・差分
4. TTS（TTSService）＋音声差し替え＋タイミング調整＋再 TTS
5. クリック強調のプレビュー合成＋インスペクタ調整
6. タイムライン編集 — 動画の指定区間の削除（範囲選択して映像ごと削除し後続を詰める）、
   セグメントの結合・分割、トリム
7. 書き出し（ExportService）
8. 初回ウィザード・設定（VOICEVOX 自動取得、API キー、話者）・インストーラ

## 非対象（YAGNI）

- 外部動画ファイルの読み込み・画像解析によるクリック検出（録画はアプリ自身で行う）
- 本格 NLE（複数トラック、自由なクリップ配置、トランジション）
- カーソルの常時追跡表示（クリック点のみ）
- macOS / Linux 対応（当面 Windows のみ）
